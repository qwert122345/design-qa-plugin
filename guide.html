Design QA Compare 사용 가이드

Figma 디자인과 실제 구현된 안드로이드 화면을 나란히 놓고 비교하면서, 어긋난 부분을 그 자리에서 기록하고 PNG 또는 Figma 페이지로 바로 넘길 수 있는 QA 도구입니다. Figma 파일과 GitHub 저장소는 어떤 경우에도 수정하지 않습니다.

01준비물 02설치 (한 번만) 03매일 쓰는 법 04문제 해결

준비물

아래 네 가지가 갖춰져 있어야 설치를 시작할 수 있어요.

  • Figma Desktop 앱 브라우저판은 개발 플러그인 설치가 안 돼요
  • Node.js 18 이상 터미널에 node -v로 확인
  • Android platform-tools (adb) 보통 Android Studio 설치 시 함께 깔림
  • 에뮬레이터 또는 실기기 QA할 빌드가 설치되어 있고, USB 디버깅 허용된 상태

설치 (한 번만)

전달받은 zip 파일마다 한 번씩만 해두면 돼요. 코드가 새로 전달되면 04번 문제 해결 항목의 마지막 안내만 다시 하면 됩니다.

1

압축 풀기

전달받은 .zip 파일을 원하는 위치에 압축 해제합니다.

2
TERMINAL

헬퍼 서버 의존성 설치. 이 서버는 QA 중에만 로컬에서 켜두는 작은 프로그램으로, 내 컴퓨터의 adb로 에뮬레이터 화면을 찍어서 플러그인에 넘겨주는 역할만 해요.

# 압축을 푼 폴더 안에서
cd helper-server
npm install
3
TERMINAL

플러그인을 빌드합니다.

cd figma-plugin
npm install
npm run build
4

Figma Desktop 우측 상단 메뉴 → Plugins → Development → Import plugin from manifest… → figma-plugin/manifest.json 선택.

이제 Plugins → Development 목록에 "Design QA Compare"가 보일 거예요.

매일 쓰는 법

설치가 끝났다면 QA할 때마다 이 다섯 단계만 반복하면 됩니다.

구현 화면
Figma 프레임
↓ chips + 텍스트로 QA 메모 → PNG 또는 Figma로 내보내기
1

cd helper-server && npm start로 헬퍼 서버를 켜두고, Figma에서 플러그인을 실행합니다.

2

상단에서 기기를 선택합니다. 그 기기를 처음 쓴다면 "밀도 360dp 맞추기"를 한 번 눌러주세요 — 캡처되는 화면의 글자 크기가 Figma와 맞게 보정돼요 (안드로이드 설정의 "디스플레이 크기"를 조정하는 것과 같은 효과이며 언제든 "밀도 초기화"로 되돌릴 수 있어요). 위치를 잡을 땐 "실시간 보기 시작"으로 화면을 보며 이동하면 편해요.

3

Figma 캔버스에서 비교할 프레임을 1개 선택합니다. 우측에 자동으로 표시돼요. 프레임을 바꿔 선택하면 이전에 쓰던 QA 메모는 자동으로 비워집니다.

4

하단 왼쪽에서 chip(typography · color · icon · margin · spacing · radius · typo_error · component)을 고르고, 어떤 부분이 어떻게 다른지 텍스트로 적은 뒤 "항목 추가"를 누릅니다. 필요한 만큼 반복해서 쌓아두면 오른쪽에 목록으로 모여요.

5

다 됐으면 아래 두 버튼 중 하나를 누릅니다. 클릭한 순간 화면을 알아서 캡처하기 때문에 따로 "캡처" 버튼을 누를 필요는 없어요.

PNG로 내보내기 — 구현 화면과 Figma 프레임을 나란히 놓고 QA 메모를 합친 PNG 한 장이 다운로드됩니다. 슬랙 공유나 첨부용으로 좋아요.

Figma에 내보내기 — 원본 디자인 페이지는 그대로 두고, 플러그인 전용 Design QA Handoff 페이지에 구현 화면 캡처본과 선택한 프레임의 복사본, chip 형태의 QA 목록을 나란히 추가해줍니다.

문제 해결

대부분 헬퍼 서버 상태와 관련된 문제예요.

헬퍼 서버 연결 실패 (localhost:4747)

helper-server 폴더에서 npm start가 실행 중인지 확인하세요.

adb를 찾을 수 없습니다

Android platform-tools가 설치되어 있는지, PATH에 등록되어 있는지 확인하세요.

DPI 정보를 가져오지 못했습니다

괄호 안 원인(기기 미인증/오프라인 등)을 확인하세요. 이 상태에서도 내보내기는 계속 동작해요.

밀도 맞추기 실패: 요청 실패 (404 Not Found)

헬퍼 서버 코드가 업데이트된 뒤 재시작을 안 한 경우예요. 서버를 껐다가 다시 켜보세요.

코드가 새로 전달되면: 헬퍼 서버 재시작 + npm run build 후 플러그인 다시 실행 궁금한 점은 만든 사람에게 물어보세요
